Version Compatibility
バージョン互換性

This book describes Swift 4.0, which is the default version of Swift included in Xcode 9. You can use Xcode 9 to build targets that are written in either Swift 4 or Swift 3.
この本はSwift 4.0を記述します、それはXcode 9に含まれるスウィフトの省略時のバージョンです。あなたは、Xcode 9を使ってSwift 4またはSwift 3のどちらかで書かれたターゲットをビルドできます。

Note
注意

When the Swift 4 compiler is working with Swift 3 code, it identifies its language version as 3.2—this means you can use conditional compilation blocks like #if swift(>=3.2) to write code that is compatible with multiple versions of the Swift compiler.
Swift 4コンパイラがSwift 3コードを扱う場合、それはその言語バージョンを3.2と識別します — これはあなたが#if swift(>=3.2)のような条件コンパイルブロックを使って複数のバージョンのスウィフトコンパイラと互換性のあるコードを記述できるのを意味します。

When you use Xcode 9 to build Swift 3 code, most of the new Swift 4 functionality is available. That said, the following features are available only to Swift 4 code:
あなたがXcode 9を使ってSwift 3コードをビルドする場合、新しいSwift 4機能性のほとんどが利用可能です。とはいえ、以下の機能はSwift 4コードでのみ利用可能です：

  • Substring operations return an instance of the Substring type, instead of String.
    添え字操作がSubstring型のインスタンスを返す、Stringではなく。

  • The @objc attribute is implicitly added in fewer places.
    @objc属性がいくらかの場所で暗黙的に加えられる。

  • Extensions to a type in the same file can access that type’s private members.
    同じファイル中のある型への拡張は、その型のもつ非公開メソッドメンバにアクセスできる。

A target written in Swift 4 can depend on a target that’s written using Swift 3, and vice versa. This means, if you have a large project that is divided into multiple frameworks, you can migrate your code from Swift 3 to Swift 4 one framework at a time.
Swift 4で書かれるあるターゲットは、Swift 3を使って書かれるターゲットを当てにでき、そして逆も同じです。これは、あなたが複数のフレームワークへと分割される巨大なプロジェクトを持つならば、あなたはあなたのコードをSwift 3からSwift 4同等フレームワークへと一度に移行できるのを意味します。
